Security

Managing API keys

Every workspace can hold up to five active API keys. Keys are scoped to a single environment and can be rolled at any time from the Settings panel.

When you roll a key, the previous key remains valid for 24 hours to allow a clean migration. After that window it is revoked automatically.

Store keys as environment variables — never commit them to source control.

How do I reset my API key without breaking my integration?
Roll the key from Settings → API Keys. Your old key keeps working for 24 hours, so you can update your integration first and then let the old key expire.
